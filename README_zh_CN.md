# 思源笔记配置同步插件

通过云端同步，在不同设备间同步思源笔记配置。

## 功能特性

- **保存**当前思源配置（编辑器、快捷键、外观、文件树、搜索、标签、导出、闪卡、AI 设置）为命名配置
- **应用**已保存的配置到任意其他设备，即时复制设置
- **平台标签** — 配置按平台标记（Windows、macOS、Linux、Android、iOS、HarmonyOS、Docker），跨平台应用时显示兼容性警告
- **选择性同步** — 自由选择需要保存或应用的配置模块
- **应用前备份** — 覆盖设置前可选择自动创建备份
- **重命名、更新、删除**配置 — 通过可视化管理界面操作
- **按平台筛选**配置列表
- **多语言**支持（中文、英文）

## 工作原理

插件将配置文件以 JSON 格式保存在 `data/storage/petal/siyuan-plugin-settings-sync/` 目录下。由于思源笔记会自动将 `data/` 目录同步到云端，您的配置会自动出现在所有设备上。

**无需额外的云端配置** — 只要启用了思源的内置同步功能，配置即可自动同步。

## 多工作空间同步（本机）

如果你在同一台电脑上有多个思源工作空间，本插件可以借助思源内核的 `globalCopyFiles` API 在它们之间**直接通过磁盘**共享配置，不依赖云端。

在管理界面中：

- 顶部新增**工作空间**下拉，可切换查看其他工作空间下的配置列表。
- 远端配置卡片提供 **拉取**（复制到当前工作空间）和 **直接应用**（拉取并立即应用）两个动作。
- 本地配置在 ⋯ 菜单中新增 **推送到…**，可一次推送到多个目标。
- 在**插件设置**中可选填**共享目录**（任意绝对路径）作为各工作空间之间的中枢，并可开启**保存时自动推送**。

注意事项：

- 该功能仅在桌面端可用。在 iOS / Android / 部分 Docker 等沙箱平台上，内核会拒绝跨工作空间路径，首次失败后插件会自动隐藏工作空间选择器。
- 不需要其他工作空间的思源进程在运行 —— 内核在本机磁盘上完成读写。
- 云同步仍然照常工作；该功能是在云同步之外的额外能力。

## 使用方法

1. 点击顶部工具栏的 ⚙️ 图标（或使用命令面板："打开配置同步管理"）
2. **保存**：点击"保存当前配置"，为配置命名，选择模块，确认保存
3. **应用**：在其他设备上打开管理面板，点击配置的"应用"按钮
4. **更新**：点击"更新"用当前设置覆盖已有配置
5. **重命名/删除**：直接在列表中管理配置

## 存储结构

```
data/storage/petal/siyuan-plugin-settings-sync/
└── profiles/
    ├── {uuid}.json        # 各配置快照
    └── ...
```

## 配置模块

| 模块 | 说明 |
|------|------|
| 编辑器设置 | 编辑器偏好（字体、行高等） |
| 快捷键 | 键盘快捷方式映射 |
| 外观设置 | 主题、模式、UI 偏好 |
| 文件树设置 | 文件树显示设置 |
| 搜索设置 | 搜索行为配置 |
| 标签设置 | 标签管理设置 |
| 导出设置 | 导出偏好 |
| 闪卡设置 | 闪卡功能设置 |
| AI 设置 | AI 提供商和模型设置 |

## 开发

1. 克隆本仓库
2. 运行 `pnpm install`
3. 运行 `pnpm run dev` 进行开发（监听文件变更）
4. 运行 `pnpm run build` 生产构建（生成 `package.zip`）
